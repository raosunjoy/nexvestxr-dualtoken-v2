# Contributing to NexVestXR

We welcome contributions to the NexVestXR platform! Please follow these guidelines to ensure a smooth contribution process.

## How to Contribute
1. **Fork the Repository**: Create a fork of the repository on GitHub.
2. **Clone Your Fork**:
   ```bash
   git clone https://github.com/your-username/nexvestxr-platform.git
   ```
3. **Create a Branch**:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make Changes**: Implement your feature or bug fix.
5. **Run Tests**:
   ```bash
   ./scripts/run-tests.sh
   ```
6. **Commit Changes**:
   ```bash
   git commit -m "Add your feature description"
   ```
7. **Push to Your Fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
8. **Create a Pull Request**: Submit a pull request to the main repository.

## Code Style
- **JavaScript**: Follow Airbnb style guide (enforced via ESLint in backend).
- **Python**: Adhere to PEP 8 (enforced via flake8 in AI service).
- **Solidity**: Follow the official Solidity style guide.
- **Commits**: Use clear, concise commit messages (e.g., "fix: resolve tokenization bug").

## Issues and Features
- Check the issue tracker for existing issues or to create new ones.
- For feature requests, open an issue with a detailed description.

## Review Process
- Pull requests will be reviewed by the core team.
- Ensure all tests pass and code adheres to style guidelines.
- Address feedback promptly to expedite merging.

## Contact
For questions, reach out via Intercom or email at support@nexvestxr.com.
